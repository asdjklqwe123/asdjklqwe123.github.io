操作系统课设

2026年3月26日 · asdjklqwe123

switch_linux.s

这是一个经典的声明与定义分离（Declaration vs Definition）的链接过程。我们可以将其简化为四个步骤来记忆：

1. 声明 (The Promise) - C++ 侧

在 thread.h 中，C++ 代码“承诺”有一个叫 SWITCH 的函数存在，虽然它自己并没有写函数体。

  • 使用 extern "C" 告诉编译器：这个函数的名字不要乱改（不要进行 C++ Name Mangling），保持原样叫 SWITCH。
  • 代码位置：thread.h 
    extern "C" {
    void SWITCH(Thread *oldThread, Thread *newThread);
}

2. 调用 (The Call) - C++ 侧

在 scheduler.cc 中，Scheduler::Run 函数像调用普通函数一样调用了 SWITCH。

  • 此时编译器只知道名字和参数，它会在生成的 .o 文件里留下一个**“待填空的洞”**（undefined reference），标记这里需要跳转到 SWITCH 的地址。
  • 代码位置：scheduler.cc 
    SWITCH(oldThread, nextThread);

3. 定义 (The Real Deal) - 汇编侧

在 switch-linux.s 中，汇编代码实现了真正的 SWITCH 逻辑，并将其标记为全局符号。

  • .globl SWITCH：告诉汇编器，“把 SWITCH这个标签暴露给外部世界使用”。如果没有这一行，C++ 代码是找不到它的。
  • SWITCH:：这是汇编函数的入口地址标签。
  • 代码位置：switch-linux.s 
        .globl SWITCH
SWITCH:
    # ... 保存 oldThread 寄存器 ...
    # ... 恢复 newThread 寄存器 ...
    ret

4. 链接 ( The Handshake) - 构建阶段

最后，make 调用链接器（ld）将所有 .o 文件合并成可执行文件 nachos。

  • 链接器发现 scheduler.o 有一个调用 SWITCH 的“空洞”。
  • 链接器发现 switch-linux.o 提供了一个叫 SWITCH 的地址。
  • 链接器把“空洞”填上 SWITCH 的实际地址。

简记公式：

C++头文件声明 (extern “C”) + 汇编文件定义 (.globl) -> 链接器配对 (Linker resolves symbol)

扩展 -ap OpenFile::WriteAt 中的注释功能实现

  1. 修改 FileHeader 类 (filehdr.h/filehdr.cc)：添加了 Extend 方法，用于在需要时分配新的磁盘扇区并更新文件头信息。
  2. 修改 FileSystem 类 (filesys.h)：公开了 GetFreeMapFile() 方法，允许其他模块获取空闲块位图文件，以便在分配新扇区时读取和更新位图。
  3. 修改 OpenFile 类 (openfile.h/openfile.cc)：
    • 在构造函数中记录了文件头的扇区号 headerSector，以便在其变化（如文件大小时）写回磁盘。
    • 在 WriteAt 中实现了自动扩展逻辑：当写入位置超出当前文件大小时，会调用 Extend 尝试扩展文件。成功扩展后，会立即将更新后的文件头和空闲块位图写回磁盘，然后继续执行写入操作。如果扩展失败（如磁盘已满或超出最大文件大小），则保持原有的截断行为。
