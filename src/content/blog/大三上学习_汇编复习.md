## 汇编器、链接器、调试器、编译器与解释器程序

汇编器：将汇编语言源程序翻译为机器语言程序
链接器：将机器语言程序中间文件和系统运行时库链接，生成可执行的机器语言程序

解释器：直接在机器上解释和执行高级语言源程序


## 指令

- 3 种寻址方式

### 数据传送指令 （如 MOV、ADD）
**立即数寻址**
- 只能用于源操作数（src），不能用于目的操作数（dst）。
用途 
- 给**寄存器**赋固定值或者初值。

**寄存器寻址**
- 任何操作数不能是IP、FLAGS，**目的操作数不能是 CS**。（有专用指令）（那可以 SS？）
- 在8086中，字节寄存器只有AH、AL、BH、BL、CH、CL、DH、DL。

**存储器寻址**
- 无法编码两个存储器，因此存储器操作数只能出现一个，要么是源，要么是目的
1. **直接寻址/绝对寻址**
	- 指令中仅含有一个16位地址偏移量，逻辑地址就是该偏移量。
	- 根据立即数作为偏移量，DS:\[IMM\]
	- 例：MOV AX, DS:\[01H\]
	- 又例：MOV BYTE PTR DS:\[0100H\], 01H

2. **间接寻址**
	- 寄存器的内容作为偏移量，DS:\[REG\]，如 REG = 01H，则取 DS:\[01H\]
	- 该寄存器必须是BX、SI、DI、BP中的一个，其它寄存器不可以。**2B2I**
	- 如果寄存器是BX、SI、DI，默认的段寄存器是 DS。
	- 如果寄存器是BP，默认的段寄存器是 SS。
	- 例：MOV AL, \[BX\]

3. **相对寻址**
	- 取 16 位寄存器的值和8位或16 位立即数的和作为偏移量
	- 间接寻址的寄存器限制和默认段寄存器在此仍然适用。(**2B2I**)
	- 8位有符号偏移量范围为 -128~127；16位无符号偏移量范围为0~65535。
	- 例：ADD \[SI+1050H\], 0102H
	
4. **基址变址寻址**
	- 指令同时指明两个 16 位寄存器，逻辑地址为两者之和。
	- 基址寄存器只要是BP，那么默认的段寄存器就是SS，否则是DS。
	- 额外限制：两个寄存器必须分别是基址寄存器（BP或BX）和变址寄存器（SI和DI）
	- 例：ADD \[BP+DI\], 0102H
	
5. **相对基址变址寻址**
 - 基址变址寻址的寄存器限制、段寄存器限制在此仍然适用
 - 偏移量的范围则与相对寻址相同。

### 控制转移寻址
- 无立即数寻址和寄存器寻址
- 只有存储器寻址，毕竟最终都要跳转到地址空间中
**段内直接寻址**
- 指令中仅含有一个 8 位或 16 位地址偏移量，目标段内地址就是该偏移量。
- SHORT：明确指出这是一个短转移，地址偏移量就在-128~127。
- NEAR PTR：明确指出这是一个近转移，地址偏移量就在-32768~32767。
- SHORT 和 NEAR PTR **可以不写**，汇编器会自行推断。SHORT 不带 PTR！
- 指令的机器码实际含有的是目标段内地址与当前 IP 的差值，跳转时用该差值加上当前 IP 值才得到指令中编码的偏移量。
**段内间接寻址**
- 例：JMP \[BX+SI+02H\]
**段间直接寻址**
- 指令中仅含有一个16位地址偏移量和一个16位段寄存器值，目标段间
- 地址就是（段寄存器值×16）+地址绝对值。
- 地址写成 段寄存器值:地址绝对值 ，由于小端字节序，存储时地址绝对值反在前。
- FAR PTR：明确指出这是一个远转移，必须写，且不能写成 DWORD PTR。
**段间间接寻址**
- 例：JMP DWORD PTR \[BX+SI+02H\]
- DWORD PTR 不可省略否则变成段内间接寻址



### 各种指令注意事项

1. **MOV** 无法移动立即数到段寄存器。（联想：寄存器寻址中目的操作数不能是 CS）
2. **PUSH** 指令
	1. `SP ← SP - 2  SS:[SP] ← 源;` 说明了栈从高到低地址生长，且是满栈
	2. PUSH 仅接受字操作。不可以使用 **IP**、FLAGS。
3. **POP** 仅接受字操作。不可以使用 **CS、IP**、FLAGS。
4. **LEA** 和MOV指令不同，reg中并不会变成\[mem\]（值），而是变成mem（值所在的地址）
5. **INC** 相当于 ADD \[mem\](或 reg), 01H，存储器寻址时要加上 **PTR**
6. **DEC** 自减指令，同 INC
7. **CMP** 原理是 **SUB**，源操作数 1 不能为立即数
8. **MUL** 
	1. 如果 源是8位：AX ← AL ×源;
	2. 否则：DX:AX ← AX ×源;
	3. 乘法指令**无法使用立即数**。
	4. 影响 **OF, CF**：结果的长度比操作数长（高半部分不为 0），则置位 OF 和 CF。
9. **DIV**
	1. 除 0 异常返回 INT 0
	2. 如果源是8位：AL ← AX / 源; AH ← AX % 源;
	3. 否则 AX ← DX:AX / 源; DX ← DX:AX % 源;
	4. 不影响标志位 FLASS
10. **AND** OF 和 CF 被**清零**，SF、ZF 和 PF 依照结果而定。其它逻辑运算同理
11. **TEST** 按 AND 指令的方法做按位与
12. **SHL**
	1. 如果最后一个移出的位是1，则 CF 置1；
	2. 如果 **1 位位移**后导致符号变化，则 OF 置1；
	3. SF、ZF 和 PF 依照结果而定。
13. **SAL** 和 SHL 其实是**同一条指令**的不同助记符写法
14. **SHR**
	1. 如果最后一个移出的位是 1，则 CF 置1；
	2. 如果 1 位位移后导致符号变化，则OF置1，事实上这只可能发生在负数到正数的情况
	3. SF、ZF和PF依照结果而定。
15. **SAR** 与 SHR 是货真价实的两条指令
16. **JMP** 对于短跳转和近跳转，JMP指令的下一条指令的起始地址+JMP指令的机器码中编码的立即偏移量。其它跳转方式都是地址绝对值。
17. 所有的 **条件跳转** 都是 **短跳转**。
18. **CALL** 先 CS 再 IP

**汇编语言** 由助记符和操作数组成的汇编码作为基本元素，直接描述计算机硬件基本操作序列（指令流）的编程语言

**时空局部性**	
程序有时空局部性，也即总是在某个时间段集中访问同一段存储空间。


**8086 六类指令**。
数据传送、算术运算、位操作、串操作、控制转移、处理机控制

**分支预测优化**：
- 如果程序员知道哪个分支成立的几率比较小，就把它写成前向跳转，否则就把它写成后向跳转或者是不跳转。这可以增加处理器成功预测分支的概率。

**尾递归优化**	Tail Recursion Optimization
- 当过程返回前的最后一个操作是调用它自己时，由于其栈框中没有任何有效信息，因此可以在对自己的下一次调用中复用。		
- 这样，递归过程相当于在反复利用同一个栈框，不再消耗额外的栈。再对程序稍加修改，就可以将其从递归变成循环。
- 事实上，过程是比循环更强大的抽象，因为循环的实质就是尾递归。这一点在Lisp等编程语言中体现非常明显，而现代的C编译器也均支持尾递归优化。

**中断服务程序**	Interrupt Service Routine (ISR)
- 由CPU在响应中断时，自动调用的一段对应于中断号的特殊过程。		
-  也叫中断例行程序、中断向量（Interrupt Vector）。

**中断向量表**
- 对于向量号N，IP存放在[0000:4N]，CS则存放在[0000:4N+2]

**为何要给INT3分配特殊的单字节编码**
- 调试器会将断点位置换成 INT 3
- 如果目标指令本身就是单字节指令，双字节编码会导致覆盖到下一条指令的第一个字节
- 当移除断点时只会恢复第一条指令而无法恢复第二条指令导致错误
- 单字节编码可以精确地替换**任何指令的第一个字节**，而不会影响后续指令。

**系统调用约定**	
- 向系统调用传递参数、取回返回值的方法，寄存器的保护方法，以及栈的调整方法。
- 这个调用约定是由操作系统编写者决定的，应用程序的编写者必须遵循。

**键盘中断服务程序**
工作流程
- 从键盘接口读取操作键的扫描码（8255的输入端口61H）
- 将扫描码转换成字符码；
	- 大部分键的字符码为ASCII码。		 
	- 无ASCII码键（如组合键Shift、Ctrl等）的字符码为0。
	- 还有一些非ASCII码键产生一个指定的动作。
- 将键的扫描码、字符码存放在键盘缓冲区KB_BUFFER。











### 要知道的名词
1. **分支合并**：如果有两个判断的条件是一样的，那么可以把一个分支的东西全都提到另一个分支里面去，这样可以少做一次判断。
2. **子表达式合并**：有时，两个判断的条件是不同的（如X+Y>5和X+Y<3），但是条件内部含有一样的子表达式（如X+Y）。可先求子表达式的值，再在判断中使用该值，避免反复计算。
3. **分支不变外提**：有时，两个互斥分支中会有同样的操作。将它们提到分支之外可以减小汇编指令的数量，减少存储器消耗，提高指令缓存的性能。

4. **强度折减**：
	1. 如果一个操作可以用代价更低的操作等效完成，那就用代价更低的操作。
	2. 一般认为，位操作比加减法代价低，而加减法又比乘除法代价低。
	3. 例：用位移代替除2除法；用 XOR 清零寄存器的代价比MOV低
5. **循环强度折减**	Loop Strength Reduction
	1. **归纳变数**：每次循环都会规律地改变（如加上、减去、乘以或除以）某个固定常量的值。
	2. 以上一次循环产生的归纳变数为基础，通过叠加一个**小的低强度运算**得到本次循环需要的归纳变数。

		
6. **寄存器变量**
	1. 有些临时变量无需在内存中分配：如果寄存器还放得下，就干脆把它们放到寄存器去。
	2. 节约访存时间；节约内存；减小了数据缓存和数据通路的压力；可生成更精简的代码从而提高指令缓存的性能。
	3. 编译器比人擅长此优化，因为它们使用全局寄存器着色等算法。

7. **分支预测优化**：
	- 分支预测器是一个**硬件电路**
	- 一个最简单的实现（静态分支预测）是，总是预测前向跳转（IP增加的方向）不成立，后向跳转（IP减少的方向）成立，取指时从预测方向处预取。
	- 大多数程序包含循环，而循环正是后向跳转。在循环退出条件判断中，大多数时候都是不成立的（也即跳到循环开头、继续循环），只有退出循环那一次退出条件才成立，因此预测前向跳转成立能消除循环运行时的跳转损失。
	- **分支预测优化**：如果程序员知道哪个分支成立的几率比较小，就把它写成前向跳转，否则就把它写成后向跳转或者是不跳转。这可以增加处理器成功预测分支的概率。

8. **循环旋转**
	- 对于那些需要在开头检查条件的while型循环，可以把第一次条件检查单独拿出来进行，然后再进入一个do-while型循环。
	 ![[Pasted_image_20260102104339.png]]
9. **循环倒转**
	- 对于那些递增循环，在不破坏数据依赖的前提下可以将它们写成递减循环，这有利于利用部分处理器的**内置循环指令**以提高性能。
10. **数据转置**	Data Transposition
	- 当被反复使用的核心数据与其它数据的行列存储方式不一致时，可以提前一次性将其转置
